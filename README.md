    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a name="user-content-getting-and-cleaning-data-peer-assessment-project" class="anchor" href="#getting-and-cleaning-data-peer-assessment-project" aria-hidden="true"><span class="octicon octicon-link"></span></a>Getting and Cleaning Data, Peer Assessment Project</h1>

<p>The purpose of this project is to demonstrate your ability to collect, work with, and clean a data set. The goal is to prepare tidy data that can be used for later analysis. You will be graded by your peers on a series of yes/no questions related to the project. You will be required to submit: 1) a tidy data set as described below, 2) a link to a Github repository with your script for performing the analysis, and 3) a code book that describes the variables, the data, and any transformations or work that you performed to clean up the data called CodeBook.md. You should also include a README.md in the repo with your scripts. This repo explains how all of the scripts work and how they are connected. </p>

<p>One of the most exciting areas in all of data science right now is wearable computing - see for example this article . Companies like Fitbit, Nike, and Jawbone Up are racing to develop the most advanced algorithms to attract new users. The data linked to from the course website represent data collected from the accelerometers from the Samsung Galaxy S smartphone. A full description is available at the site where the data was obtained:</p>

<p><a href="http://archive.ics.uci.edu/ml/datasets/Human+Activity+Recognition+Using+Smartphones">http://archive.ics.uci.edu/ml/datasets/Human+Activity+Recognition+Using+Smartphones</a></p>

<p>Here are the data for the project:</p>

<p><a href="https://d396qusza40orc.cloudfront.net/getdata%2Fprojectfiles%2FUCI%20HAR%20Dataset.zip">https://d396qusza40orc.cloudfront.net/getdata%2Fprojectfiles%2FUCI%20HAR%20Dataset.zip</a></p>

<p>You should create one R script called run_analysis.R that does the following. </p>

<ol class="task-list">
<li>Merges the training and the test sets to create one data set.</li>
<li>Extracts only the measurements on the mean and standard deviation for each measurement. </li>
<li>Uses descriptive activity names to name the activities in the data set</li>
<li>Appropriately labels the data set with descriptive variable names. </li>
<li>Creates a second, independent tidy data set with the average of each variable for each activity and each subject. </li>
</ol><h2>
<a name="user-content-how-to" class="anchor" href="#how-to" aria-hidden="true"><span class="octicon octicon-link"></span></a>How to</h2>

<ol class="task-list">
<li>Download source data from <a href="http://archive.ics.uci.edu/ml/datasets/Human+Activity+Recognition+Using+Smartphones">http://archive.ics.uci.edu/ml/datasets/Human+Activity+Recognition+Using+Smartphones</a>
</li>
<li>Unzip to your working directory</li>
<li>Execute the script run_analysis.R from that directory.</li>
</ol><h2>
<a name="user-content-steps-performed" class="anchor" href="#steps-performed" aria-hidden="true"><span class="octicon octicon-link"></span></a>Steps Performed</h2>

<div class="highlight highlight-sh"><pre>Step 1:
*At first, you should install packages of <span class="s2">"data.table"</span> and <span class="s2">"reshape2"</span>
*Loading packages
*Getting Course Project<span class="err">'</span>s data path
*Reading .txt files of <span class="s2">"train"</span> and <span class="s2">"test"</span>

Step 2:
*Merges the training and the <span class="nb">test </span>sets to create one data set.
2-1: Concatenate the data tables.
2-2: Merge columns and <span class="nb">set </span>the key.

Step 3:
*Extracts only the measurements on the mean and standard deviation <span class="k">for</span> each measurement.

*Read the features.txt file to know what colummns are measurements <span class="k">for</span> the mean and standard deviation in dt.
*Convert the column numbers to a vector of variable names.
*Subsetting variables with variable names.

Step 4:
*Uses descriptive activity names to name the activities in the data set.
*Read activity_labels.txt file to know how to add descriptive names to the activities.

Step 5:
*Appropriately labels the data <span class="nb">set </span>with descriptive variable names.
*Merge activity labels and add activityName as a key

Step 6:
*Creates a second, independent tidy data <span class="nb">set </span>with the average of each variable <span class="k">for</span> each activity and each subject.

Step 7:
*Output tidy data file and upload it.
</pre></div></article>
